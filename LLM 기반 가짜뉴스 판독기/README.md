# LLM 기반 가짜뉴스 판독기

2025 Upstage Agentic Workflow Hackathon에서 진행한 해커톤 프로젝트입니다.  
텔레그램 봇을 입력 창으로 사용하고, `n8n` 워크플로우 안에서 `Upstage Solar`, `Google Search`, 자막 추출 흐름을 연결해 뉴스 기사와 유튜브 영상의 사실 여부를 빠르게 점검할 수 있도록 구성했습니다.

이 프로젝트는 복잡한 검증 절차를 사용자가 직접 수행하지 않아도, 링크 하나를 보내면 `수집 → 분석 → 검색 → 비교 → 회신`까지 이어지는 경험을 만드는 데 초점을 맞췄습니다.

## 프로젝트 개요

가짜뉴스와 딥페이크 콘텐츠가 빠르게 확산되면서 일반 사용자가 직접 진위를 판별하기 어려운 상황이 많아졌습니다.  
이 프로젝트는 기사 URL이나 유튜브 영상 URL을 텔레그램 봇으로 전송하면, 관련 내용을 자동으로 수집하고 검색 결과와 비교해 1차 판별 결과를 제공하는 워크플로우를 구현한 해커톤 결과물입니다.

## 내가 맡은 역할

- 뉴스 기사 내용을 받아 사실 여부를 판별하는 핵심 기능 흐름을 구현했습니다.
- 기사 본문에서 주요 내용을 추출하고, 검색 결과와 비교해 진위 여부를 정리하는 판단 로직을 맡았습니다.
- 사용자가 결과를 이해하기 쉽도록 텔레그램 회신 메시지 흐름도 함께 구성했습니다.

## 핵심 기능

### 1. 뉴스 기사 팩트체킹

- 텔레그램으로 기사 URL 입력
- 기사 본문 HTML 수집 및 텍스트 추출
- Upstage Solar를 활용한 검색용 키워드 생성
- Google Search 결과와 원문 비교
- 텔레그램으로 판별 결과 회신

### 2. 유튜브 영상 팩트체킹

- 텔레그램으로 유튜브 URL 입력
- `yt-dlp`를 활용한 한국어 자막 추출
- 자막 내용을 바탕으로 검색 쿼리 생성
- 검색 결과와 원본 내용을 비교해 사실 여부 판단
- 텔레그램으로 판별 결과 회신

## 사용 기술

- `n8n`
- `Telegram Bot`
- `Upstage Solar`
- `Google Search API (SerpApi)`
- `JavaScript Code Node`
- `Python`
- `yt-dlp`

## 구현 포인트

- 기사와 영상을 각각 다른 워크플로우로 분리해 입력 형태에 맞는 처리 흐름을 설계했습니다.
- 단순 요약이 아니라, 검색 결과와 원문 내용을 비교해 판별 근거를 만들도록 워크플로우를 구성했습니다.
- 텔레그램을 입출력 채널로 사용해 비개발자도 쉽게 사용할 수 있는 형태를 목표로 했습니다.
- 해커톤 환경에 맞춰 짧은 시간 안에 바로 시연 가능한 구조로 구현했습니다.

## 프로젝트 의미

이 프로젝트는 LLM과 워크플로우 자동화 도구를 실제 문제 해결에 연결해본 경험이라는 점에서 의미가 있습니다.  
특히 해커톤 환경에서 `아이디어를 빠르게 구조화하고, 실제로 동작하는 검증 흐름으로 구현하는 과정`을 경험했다는 점이 이후 자동화 프로젝트와 AI 활용 프로젝트를 확장하는 데 도움이 되었습니다.
